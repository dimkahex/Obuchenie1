# Пошагово: связать проект с GitHub

У тебя уже есть **GitHub Desktop** и **репозиторий на GitHub**. Дальше — по шагам.

---

## Шаг 1. Открыть проект в GitHub Desktop

1. Запусти **GitHub Desktop**.
2. Меню **File** → **Add local repository** (или **Добавить локальный репозиторий**).
3. Укажи папку проекта — та, где лежит `README.md` (папка **АУ**).
4. Если программа напишет «This directory does not appear to be a Git repository» — нажми **Create a repository** и создай репозиторий в этой папке (имя можно оставить как есть, локально).

---

## Шаг 2. Первый коммит (если ещё не было)

1. Слева в GitHub Desktop должны появиться все изменённые файлы.
2. Внизу слева в поле **Summary** напиши, например: `Первый коммит`.
3. Нажми синюю кнопку **Commit to main** (или **Commit to master**).

---

## Шаг 3. Связать с репозиторием на GitHub

1. Меню **Repository** → **Repository settings** (или **Настройки репозитория**).
2. В разделе **Primary remote repository** нажми **Add** (или вкладка **Remote**).
3. В поле **URL** вставь адрес своего репозитория с GitHub, например:  
   `https://github.com/ТВОЙ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ.git`
4. Сохрани (OK / Save).

Либо: меню **Repository** → **Push origin** — если репо ещё не связан, GitHub Desktop может предложить **Publish repository**; тогда выбери свой аккаунт и имя репозитория и опубликуй.

---

## Шаг 4. Отправить код на GitHub (Push)

1. Вверху нажми **Push origin** (или **Publish repository**, если это первый раз).
2. Если попросит — войди в GitHub (логин/пароль или токен).
3. Дождись окончания отправки.

Готово: код проекта теперь на GitHub.

## Если не получилось — делаем через командную строку

GitHub Desktop иногда путает с путями или remote. Ниже вариант через терминал (в Cursor: **Terminal → New Terminal** или `Ctrl+`` `).

### 1. Открой терминал в папке проекта

В Cursor открой папку **АУ** (File → Open Folder → твоя папка АУ). Потом открой терминал — он уже будет в корне проекта.

### 2. Проверь, есть ли Git

Введи и нажми Enter:

```
git --version
```

Если команда не найдена — установи Git с https://git-scm.com/download/win и перезапусти Cursor.

### 3. Инициализация (если в папке ещё нет Git)

По очереди выполни:

```
git init
git add .
git status
```

`git status` покажет список файлов — они должны быть зелёными (готовы к коммиту).

### 4. Первый коммит

```
git commit -m "Первый коммит"
```

### 5. Привязать твой репозиторий с GitHub

**Замени** `ТВОЙ_ЛОГИН` и `ИМЯ_РЕПО` на свои (адрес репо виден на странице репозитория на GitHub — кнопка зелёная **Code**, скопируй URL):

```
git remote add origin https://github.com/ТВОЙ_ЛОГИН/ИМЯ_РЕПО.git
```

Если напишет «remote origin already exists» — тогда сначала:

```
git remote remove origin
```

и снова команду с твоим URL.

### 6. Отправить код на GitHub

```
git branch -M main
git push -u origin main
```

Попросит войти в GitHub (логин и пароль; для пароля может понадобиться **Personal Access Token** вместо обычного пароля — создаётся в GitHub: Settings → Developer settings → Personal access tokens).
